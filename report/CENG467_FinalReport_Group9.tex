\title{Unsupervised Neural Machine Translation for Turkish and English: Denoising Autoencoder and Iterative Back-Translation}
\author{Group 9}
\institute{CENG467 Term Project}
\maketitle

\begin{abstract}
This report details the design, implementation, and evaluation of an Unsupervised Neural Machine Translation (UNMT) system for the Turkish-English language pair, developed without using any parallel corpora. We leverage a shared lightweight Transformer encoder-decoder architecture (46M parameters) trained initially as a Denoising Autoencoder (DAE) to model language-specific distributions, followed by Iterative Back-Translation (IBT) to align the latent spaces. Our implementation builds heavily upon the methodologies proposed by Lample et al. (2018). We present extensive evaluations on the FLORES-200 benchmark, comparing our unsupervised approach against a word-by-word fastText baseline and a supervised Helsinki-NLP reference model. Detailed error analyses highlight the specific challenges posed by Turkish agglutinative morphology in an unsupervised setting.
\end{abstract}

\section{Introduction}
Neural Machine Translation (NMT) has achieved near-human parity on resource-rich language pairs by relying on massive parallel corpora. However, parallel data is inherently scarce, expensive to create, and unavailable for many language combinations. Unsupervised Neural Machine Translation (UNMT) addresses this by training models solely on monolingual corpora. In this project, we explore the challenges of applying UNMT to the Turkish-English pair, a morphologically rich and structurally distinct combination.

\section{Related Work}
Lample et al. (2018) introduced a robust framework for UNMT relying on three core principles: 1) shared latent representations, 2) denoising autoencoding, and 3) iterative back-translation. Subsequent works such as XLM and mBART extended this by pre-training massive masked language models. Due to computational constraints, our project replicates the fundamental principles of Lample et al. using a lightweight Transformer.

\section{Methodology}
\subsection{Data Preprocessing and Vocabulary}
We initially utilized 1,000,000 sentences per language from the CC-100 dataset. However, due to computational bottlenecks during the iterative back-translation phase, we later sub-sampled the dataset to 200,000 sentences per language to allow for a higher number of training iterations. We trained a joint SentencePiece Unigram vocabulary (size 32,000) on both corpora to enforce a shared embedding space. Case was preserved to avoid Turkish I/ı corruption, while Turkish-aware lowercasing remained available as an optional preprocessing mode. Aggressive web-noise filtering was applied.

\subsection{Model Architecture}
A 4-layer, 8-head shared Transformer Encoder-Decoder (46M parameters) was implemented using PyTorch. The embedding layer is shared across both languages and tied to the output projection. A language identifier token (\texttt{<TR>} or \texttt{<EN>}) prepended to the encoder input signals the target language.

\subsection{Denoising Autoencoder (DAE)}
To establish a strong language model prior, we trained the network to reconstruct clean sentences from corrupted inputs. The noise function applies word dropping ($p=0.1$), word masking ($p=0.1$), and local word shuffling ($k=3$).

\subsection{Iterative Back-Translation (IBT)}
The core translation capability emerges during the IBT phase. The model generates synthetic parallel data by translating monolingual sentences using greedy decoding. The model is then trained with a cross-entropy objective to predict the real sentence from the synthetic translation.

\section{Experimental Setup}
\subsection{Baselines}
We established three baselines: 1) A Copy Baseline, 2) A Word-by-Word translation using MUSE-aligned fastText embeddings, and 3) A fully supervised Helsinki-NLP MarianMT model.

\subsection{Training Details}
Training was conducted in Google Colab using GPU acceleration, FP16 mixed precision, and the AdamW optimizer. The pipeline was developed in multiple stages. First, a denoising autoencoder (DAE) was trained to reconstruct corrupted Turkish and English monolingual sentences. Then, iterative back-translation (IBT) was applied to generate synthetic parallel data and gradually improve cross-lingual mapping.

During development, several implementation and data issues were identified and corrected. In particular, the final pipeline uses target-language conditioning tokens during generation, balanced Turkish and English monolingual corpora, and validated SentencePiece caches. The final experiments used a balanced 200,000-sentence dataset for each language. The best scoring model was obtained after the sixth IBT iteration on this 200k dataset. An additional filtered seventh iteration was also tested to reduce source-copying in synthetic pseudo-parallel data.

\section{Results}
\subsection{Automatic Evaluation}

Evaluation was performed on the FLORES-200 devtest split, which contains 1012 sentences for each language direction. We report BLEU and chrF for Turkish-to-English (TR$\rightarrow$EN) and English-to-Turkish (EN$\rightarrow$TR). All reported scores were obtained without using any parallel training data. The supervised Helsinki-NLP system is used only as an external reference and was not used for training, back-translation, or distillation.

The best average BLEU score was achieved by the Iteration 6 model trained on the balanced 200k dataset and decoded using beam search with beam size 4. This model reached 2.63 BLEU for TR$\rightarrow$EN and 2.41 BLEU for EN$\rightarrow$TR. Beam search slightly improved BLEU compared to greedy decoding for Iteration 6, although chrF changed only marginally.

\subsection{Effect of Filtered Iterative Back-Translation}

A major issue observed during IBT was source-copying, where the model copied many source-side tokens instead of producing a fluent target-language translation. To address this, we introduced a filtered IBT step before Iteration 7. The filter removed synthetic sentence pairs with high source-target token overlap, very short hypotheses, or clearly wrong target-language outputs.

During the filtered Iteration 7 run, the filter kept 90,599 out of 200,000 TR$\rightarrow$EN synthetic pairs and 65,971 out of 200,000 EN$\rightarrow$TR synthetic pairs. This indicates that a large portion of the generated synthetic data was noisy or copy-like, especially in the EN$\rightarrow$TR direction. The filtering step improved EN$\rightarrow$TR BLEU from 2.41 to 2.50 compared with Iteration 6 using beam search, but it also reduced TR$\rightarrow$EN BLEU from 2.63 to 2.49. This shows a trade-off between metric performance and copy reduction.

\section{Error Analysis}

We performed automatic copy-like diagnostics on the generated translations. A translation was marked as copy-like when more than half of its output tokens overlapped with the source sentence. Although this heuristic is not a perfect measurement of translation quality, it is useful for detecting one of the main failure modes of unsupervised NMT: copying the input instead of translating it.

The diagnostic results confirm that the filtered IBT step substantially reduced copying. In TR$\rightarrow$EN, the copy-like ratio decreased from 36.36\% in Iteration 6 greedy decoding to 22.23\% in Iteration 7 filtered greedy decoding. In EN$\rightarrow$TR, the ratio decreased from 81.92\% to 72.92\%. This supports the conclusion that synthetic data filtering improves the quality of the training signal, even if it does not always maximize BLEU.

The remaining errors can be grouped into three main categories:
\begin{itemize}
    \item \textbf{Source-copying and mixed-language outputs:} The model often preserves source-side words, especially named entities, numbers, and common English words in EN$\rightarrow$TR translation.
    \item \textbf{Morphological errors:} Turkish agglutinative morphology creates many surface forms, which are difficult to learn from noisy monolingual data without parallel supervision.
    \item \textbf{Semantic drift and hallucination:} Some outputs are fluent but only weakly related to the source sentence, showing that the model has not fully aligned the two language spaces.
\end{itemize}

\section{Discussion and Limitations}

The experiments highlight the difficulty of training an unsupervised NMT system from scratch for a morphologically distant pair such as Turkish and English. Even after DAE pretraining, balanced data preparation, 200k-scale IBT, beam search, and pseudo-parallel filtering, the model remains far below supervised systems. This is expected because supervised models are trained on large parallel corpora and often benefit from pretraining, while our model only uses monolingual data.

A key observation is that automatic BLEU improvement and actual translation quality do not always move together. Iteration 6 with beam search achieved the best average BLEU, but it also had high copy-like ratios. Filtered Iteration 7 reduced copying significantly and improved EN$\rightarrow$TR BLEU, but slightly decreased TR$\rightarrow$EN BLEU. Therefore, the final system should be interpreted through both automatic metrics and error diagnostics.

The main limitations of this work are:
\begin{enumerate}
    \item \textbf{Limited model and data scale:} The Transformer model is lightweight compared with modern multilingual NMT systems, and the final dataset contains 200k monolingual sentences per language.
    \item \textbf{No multilingual pretraining:} The system learns cross-lingual alignment only from DAE and IBT objectives. Modern UNMT systems usually rely on strong cross-lingual pretraining such as XLM, mBART, or mT5.
    \item \textbf{No true parallel supervision:} The absence of parallel training data makes semantic alignment unstable, especially for distant languages.
    \item \textbf{Noisy web text:} CC-100 contains noisy, domain-imbalanced, and sometimes incomplete web sentences. This affects both language modeling and synthetic data quality.
\end{enumerate}

Despite these limitations, the project successfully demonstrates a full UNMT pipeline: monolingual preprocessing, shared vocabulary construction, DAE pretraining, iterative back-translation, synthetic data caching, beam-search evaluation, and pseudo-parallel filtering.

\section{Conclusion}

This project implemented and evaluated an unsupervised Turkish-English neural machine translation system using denoising autoencoding and iterative back-translation. The final score-best model was Iteration 6 trained on a balanced 200k monolingual dataset and decoded with beam search, achieving 2.63 BLEU for TR$\rightarrow$EN and 2.41 BLEU for EN$\rightarrow$TR on FLORES-200 devtest. A filtered Iteration 7 experiment reduced source-copying substantially and achieved the best EN$\rightarrow$TR BLEU score of 2.50, but slightly reduced TR$\rightarrow$EN performance.

Overall, the results show that UNMT from scratch is possible to implement but remains highly challenging for Turkish-English under limited computational resources. The most important improvement for future work would be to initialize the model with large-scale multilingual pretraining or to use stronger cross-lingual representations before iterative back-translation.
